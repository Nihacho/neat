## 1. Introducción
**Gestión de Activos (Neat)** es un sistema integral diseñado para el control eficiente de activos institucionales (Universidad "X"). Permite administrar el ciclo de vida completo de los activos, desde su registro y ubicación hasta su asignación y control de préstamos a docentes, funcionarios y estudiantes.

### Tecnologías Principales
El proyecto está construido sobre un stack moderno y eficiente:
- **Frontend:** React 19 (UI Library)
- **Compilador/Entorno:** Vite (Rápido y ligero)
- **Estilos:** Tailwind CSS 4 (Utility-first CSS)
- **Base de Datos:** Supabase (PostgreSQL as a Service)
- **Estado/Datos:** React Query (TanStack Query)
- **Navegación:** React Router 6+
- **Iconografía:** Lucide React

## 2. Instalación y Configuración

### 2.1 Requisitos Previos
Para ejecutar este proyecto en un entorno local, asegúrese de tener instalado:
- **Node.js**: Versión 20.19+ o 22.12+ (Recomendado).
- **Gestor de paquetes**: npm (incluido con Node.js) o yarn.
- **Cuenta de Supabase**: Para la base de datos y autenticación.

### 2.2 Pasos de Instalación

1.  **Clonar el repositorio:**
    Descargue el código fuente en su máquina local.
    `cd ruta/a/tu/carpeta/neat`

2.  **Instalar dependencias:**
    Ejecute el siguiente comando para descargar todas las librerías necesarias.
    `npm install`

3.  **Configurar Variables de Entorno:**
    Cree un archivo llamado `.env` en la raíz del proyecto. Copie el siguiente formato y reemplace con sus credenciales de Supabase:
    ```
    VITE_SUPABASE_URL=https://tu-proyecto.supabase.co
    VITE_SUPABASE_ANON_KEY=tu-anon-key-publica
    ```

4.  **Base de Datos (Migraciones):**
    Ejecute los scripts SQL proporcionados en la carpeta `database/` en su panel de Supabase:
    - `create_usuario_table.sql`
    - `add_cantidad_to_activo.sql`
    - `enhance_prestamo_table.sql`

5.  **Ejecutar en Desarrollo:**
    Inicie el servidor local: `npm run dev`
    Acceda a: `http://localhost:5173`

6.  **Construcción para Producción:**
    Para generar los archivos optimizados: `npm run build`

## 3. Arquitectura del Proyecto

-   **`assets/`**: Archivos estáticos como imágenes y logotipos.
-   **`components/`**: Componentes de UI reutilizables (Botones, Modales, Badges).
-   **`layouts/`**: Estructuras de página principales (Sidebar, Header).
-   **`pages/`**: Vistas principales (Dashboard, Activos, Personas, Préstamos).
-   **`features/`**: Lógica de negocio agrupada por dominio (auth, assets, loans, users).
-   **`lib/`**: Utilidades y configuración de Supabase.

## 4. Base de Datos (Esquema)
-   **`activos`**: Información de bienes.
-   **`personas`**: Registro de solicitantes (Estudiantes, Docentes).
-   **`ubicaciones`**: Lugares físicos.
-   **`prestamos`**: Transacciones de activos.
-   **`users`**: Tabla personalizada de usuarios del sistema.

## 5. Seguridad
-   **Autenticación**: Supabase Auth.
-   **Roles**: Diferenciación de permisos (Admin vs Operador).

## 1. Acceso al Sistema

### Iniciar Sesión
1.  Abra su navegador e ingrese la dirección del sistema.
2.  En la pantalla de inicio, ingrese su **Correo Electrónico** y **Contraseña**.
3.  Haga clic en **"Iniciar Sesión"**.

## 2. Panel Principal (Dashboard)
Esta pantalla ofrece un resumen rápido del estado de la universidad:
-   **Tarjetas Superiores:** Total de activos, préstamos y alertas.
-   **Gráficos:** Distribución de activos.
-   **Accesos Rápidos:** Botones para acciones frecuentes.

## 3. Gestión de Activos

### Buscar un Activo
1.  Vaya a la sección **Activos**.
2.  Use la barra de búsqueda para filtrar por nombre, código o serie.

### Registrar Nuevo Activo
1.  Haga clic en **"+ Nuevo Activo"**.
2.  Complete el formulario (Nombre, Código, Serie, Estado, Ubicación).
3.  Haga clic en **Guardar**.

### Editar o Eliminar
-   **Editar**: Ícono de lápiz.
-   **Eliminar**: Ícono de basura (requiere confirmación).

## 4. Gestión de Préstamos

### Registrar un Préstamo (Salida)
1.  Vaya a **Préstamos**.
2.  Haga clic en **"Nuevo Préstamo"**.
3.  Seleccione el **Activo** y la **Persona**.
4.  Indique la **Fecha de Devolución**.
5.  Haga clic en **Confirmar Préstamo**.

### Registrar Devolución (Entrada)
1.  Busque el préstamo activo en la lista.
2.  Haga clic en **"Devolver"**.
3.  Verifique el estado y confirme.

## 5. Gestión de Personas
Puede registrar tres tipos de perfiles:
-   **Docentes**
-   **Funcionarios**
-   **Estudiantes**
Asegúrese de ingresar correctamente el Documento de Identidad y Correo.

## 6. Gestión de Ubicaciones
Defina la estructura física para organizar activos:
-   Bloques / Edificios
-   Pisos
-   Ambientes / Salones

## 7. Reportes
Visualice listados para auditoría, filtre por fechas o estado, y exporte la información si es necesario.
